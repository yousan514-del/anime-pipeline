## 🗓 タイムライン

### 2026年2月初旬 — 起点：「量産と品質の矛盾」

pixivにAIイラストを継続投稿するため、ComfyUI を使い始めた。
最初は手動で1枚ずつプロンプトを打ち込んでいたが、すぐに限界を感じた。

**問題点:**
- プロンプトをゼロから考えるのに時間がかかる
- 生成した画像の品質チェックが目視任せで非効率
- 「この画像、投稿していいの？」という判断が都度必要
- 同じ作業を毎日繰り返しているという非合理さ

> 「AI に AI の仕事をさせられないか？」

この問いがパイプライン開発のスタートになった。

---

### 2026年2月中旬 — `prompt_gen.py` の誕生

Ollama + Qwen 2.5-VL 7B がローカルで動くことを確認してから、
「日本語テーマ → ComfyUI英語プロンプト」の自動変換を試みた。

最初は単純なプロンプトテンプレートだったが、
Qwen に世界観（メタバース大学・AI社員・2040年の未来）を
システムプロンプトで注入することで、一貫したトーンのプロンプトが
生成されるようになった。

```python
# 初期の実装イメージ
system = "You are a ComfyUI prompt expert for anime-style illustration..."
theme  = "メタバース大学の夕暮れの教室"
→ "1girl, futuristic classroom, hologram display, golden hour light, ..."
```

---

### 2026年2月下旬 — `quality_eval.py` と `comfyui_batch_gen.py`

プロンプト自動生成ができても、生成した画像が「使えるか」の判断が手動のままだった。

**Stage 1 (PIL):** 解像度・シャープネス・コントラスト・輝度・彩度を数値化。
これだけで「明らかにダメな画像」は弾けることがわかった。

**Stage 2 (Qwen VL):** PIL だけでは「構図がおかしい」「世界観と合わない」が
わからない。Qwen に画像を見せて評価させたら、人間の判断に近い結果が出た。

2段階評価 + 自動却下で、ゴミ画像が drafts フォルダに入らなくなった。
これで初めて「生成 → 評価 → 保存」が自動化された。

---

### 2026年3月初旬 — WAN 2.2 I2V との格闘

静止画だけでなく「動く絵」が作りたくなった。
WAN 2.2 の I2V モデルを導入したが、当初は設定が難解だった。

- VRAM が 12GB では足りないと思っていたが GGUF Q8 で何とかなると気づいた
- デュアルパス（HighNoise/LowNoise の切り替え）でキャラ崩れを防ぐ手法を発見
- Lightning LoRA（4ステップ）でリアルタイムに近い速度を実現

`i2v_batch_gen.py` の完成で、イラスト1枚から5秒動画が10-30秒で生成できるようになった。

---

### 2026年3月中旬 — 全パイプラインの統合

個別スクリプトが揃ったので、一貫したパイプラインとして整理した。

```
テーマ入力
  → prompt_gen.py    (Qwen → ComfyUI プロンプト)
  → comfyui_batch_gen.py (バッチ生成)
  → quality_eval.py  (PIL + Qwen 品質評価)
  → comfyui_sync.py  (AI Studio へ取り込み)
  → prepare_upload.py (pixiv/FANBOX 投稿パッケージ)
```

毎日の投稿作業が「テーマを決めて実行するだけ」になった。

### 2026年4月 — ポートフォリオ化

AIスタートアップへの転職を目指し、この制作物をポートフォリオとして整理。
「自分でツールを作って実運用している」という事実を見せることが
最も強いアピールになると判断した。

`portfolio_i2v.py` は SFW 専用のデモ版として新規作成。
モデルファイル名の差異（GGUF 対応、SFW LoRA のみ）を修正し、
ポートフォリオ用の examples フォルダに出力するように調整した。

## 🔧 技術的チャレンジと解決策

### VRAM 管理 (12GB RTX 4070)

| 問題 | 解決策 |
|------|--------|
| ComfyUI + Qwen の VRAM 競合 | 時間差実行（評価→生成）|
| WAN I2V で OOM | GGUF Q8 + EasyCache |
| LoRA 複数スタック時のメモリ | sequential loading |

### Qwen VL の日本語 → 英語変換精度

- システムプロンプトに詳細な世界観を注入
- `enable_thinking: false` で余計な推論テキストを除去
- `repetition_penalty: 1.05` で冗長を抑制

### ComfyUI API の安定性

- キュー送信前に `/system_stats` でヘルスチェック
- `wait_for_completion()` でポーリング（sleep 3秒）
- タイムアウト 300秒（I2V は生成に時間がかかる）
